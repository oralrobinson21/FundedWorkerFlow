Key points:

2. Make cancel request work on the Job Detail screen

Open your JobDetailScreen and add a real cancel action.

Assuming useAppContext() exposes cancelTask and you already have the task loaded:

import { Alert, TouchableOpacity, Text } from 'react-native';
import { useAppContext } from '../context/AppContext';

export const JobDetailScreen = ({ route }) => {
  const { taskId } = route.params;
  const { tasks, user, cancelTask } = useAppContext();

  const task = tasks.find(t => t.id === taskId);

  const canCancel =
    task &&
    (task.status === 'requested' || task.status === 'accepted') &&
    task.posterId === user.id;

  const handleCancel = async () => {
    Alert.alert(
      'Cancel job',
      'Are you sure you want to cancel this job?',
      [
        { text: 'No' },
        {
          text: 'Yes, cancel',
          style: 'destructive',
          onPress: async () => {
            try {
              await cancelTask(task.id);
            } catch (e) {
              Alert.alert('Error', 'Could not cancel job. Please try again.');
            }
          },
        },
      ]
    );
  };

  return (
    <>
      {/* ... existing job details ... */}

      {canCancel && (
        <TouchableOpacity
          style={{ marginTop: 16, padding: 12, backgroundColor: 'red', borderRadius: 8 }}
          onPress={handleCancel}
        >
          <Text style={{ color: 'white', textAlign: 'center' }}>Cancel Request</Text>
        </TouchableOpacity>
      )}
    </>
  );
};

Make sure your cancelTask in AppContext actually hits the backend /api/tasks/:taskId/cancel and then updates local state.

3. Double-check nothing is blocking touches

If none of the buttons on that screen work (even after adding onPress):
	•	Check that your top-level wrapper around ProfileScreen does NOT have something like:

<View pointerEvents="none">
  {/* whole screen */}
</View>

or a full-screen overlay view (like a loading spinner) with position: 'absolute' and pointerEvents="auto" that never goes away.

That will eat all touches.

The simplest safe structure is:

return (
  <View style={{ flex: 1 }}>
    {/* content, buttons, etc. */}
  </View>
);

No transparent overlays on top.
